Build Commands
First, navigate to the project directory:
cd /home/symbol/embedded_sdk_rfid/fxr90/New_testing_Backup
Then, use the following commands to build the different test suites:
-- Build regression tests (clean and debug)
make -f makeRegression linux64_clean
make -f makeRegression linux64_debug

-- Build manual tests (clean and debug)
make -f makeManual linux64_clean
make -f makeManual linux64_debug

Your compiled binaries (ELF files) will be located in the linux64_clean and linux64_debug directories.

âš™ï¸ A Closer Look at Access Operations
This suite provides deep coverage of RFID tag access operations.

|   Feature               | Description                                                                                                         |
|-------------------------|---------------------------------------------------------------------------------------------------------------------|
| **Read/Write Access**   | Utilizes structures like `READ_ACCESS_PARAMS` and `WRITE_ACCESS_PARAMS` for core operations.                        |
| **Access Sequences**    | Batch multiple operations using APIs like `RFID_InitializeAccessSequence` and `RFID_PerformAccessSequence`.         |
| **Event Handling**      | Employs semaphores and callbacks for `ACCESS_START_EVENT` and `ACCESS_STOP_EVENT` to ensure proper synchronization. |
| **Security**            | Includes tests for password management, lock/unlock operations, and authentication (G2V2, ImpinjQT, NXP).           |
| **Performance Metrics** | Tracks success/fail counts and logs detailed results for benchmarking.                                              |

Example Access Flow:
1. Initialize the Access Sequence.
2. Add Read, Write, or Lock operations to the sequence.
3. Perform the Access Sequence.
4. Handle events and log the results.
5. Stop & Deinitialize the sequence.
